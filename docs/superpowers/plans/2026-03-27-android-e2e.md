# Android Emulator E2E Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a self-contained Android emulator end-to-end test that performs the same auth/RSS/feed flow as the existing web HNRSS E2E and emits a report with crashes plus application-level warnings, errors, and failures.

**Architecture:** Add Android instrumentation support and a small diagnostics layer in `androidApp`, expose the missing stable UI selectors in the Compose UI, and add a host-side runner script that provisions an emulator, starts the fixture/server dependencies, performs a fresh install, runs the instrumentation test, captures logcat plus step progress, and writes human-readable and JSON reports.

**Tech Stack:** Android instrumentation, Compose UI test APIs, Kotlin, Gradle, adb/emulator/avdmanager/sdkmanager tooling, Node fixture server, Ktor server, shell scripting

fun signUpAddSourceSignOutSignInRefreshFlow() {
    val email = "android-e2e-${System.currentTimeMillis()}@example.com"

    step(screen = "login", name = "sign_up") {

        click("feed-add-source-fab")

        click("add-source-rss-submit-button")
        assertNoHandledErrors()
    }

    step(screen = "feed", name = "verify_initial_feed") {
        assertFeedTitles(
            "Launch HN: Deterministic Feed Fixtures",
            "Ask HN: How do you de-flake end-to-end tests?",
        )
        assertNoHandledErrors()
    }

    step(screen = "feed", name = "sign_out") {
        click("feed-sign-out-button")
        assertNodeExists("login-submit-button")
        assertNoHandledErrors()
    }

    step(screen = "login", name = "sign_in") {
        enterText("login-email-field", email)
        enterText("login-password-field", password)
        click("login-submit-button")
        assertNodeExists("feed-refresh-button")
        assertNoHandledErrors()

    step(screen = "feed", name = "refresh") {
        click("feed-refresh-button")
        assertNoHandledErrors()
    }

    step(screen = "feed", name = "verify_final_feed") {
        assertFeedTitles(
            "Launch HN: Deterministic Feed Fixtures",
            "Ask HN: How do you de-flake end-to-end tests?",
        )
        assertNoHandledErrors()
    }
}

  Runner responsibilities:
  - resolve Android SDK tools from standard environment locations
  - ensure a pinned AVD exists; create it if missing
  - boot a dedicated emulator and wait for `sys.boot_completed`
  - start `e2e/scripts/fixture-server.js`
  - start `cd src && ./gradlew :server:run --no-daemon`
  - wait for `/health` endpoints
  - clear logcat for the device
  - uninstall app package for a fresh install
  - run `cd src && ./gradlew :androidApp:connectedDebugAndroidTest --no-daemon`
  - collect:
    - instrumentation result XML/output
    - progress artifact written by the test
    - fresh logcat
  - derive issue entries for:
    - setup failures
    - warnings recorded by the test harness
    - handled UI errors
    - assertion failures
    - crashes
  - always tear down fixture server, Ktor server, and emulator
